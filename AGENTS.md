# AGENTS.md

## Project: 산지니 원정대 앱형 웹 프로토타입

이 문서는 AI coding agent가 따라야 하는 구현 기준을 정의한다.

본 프로젝트는 더 이상 "작은 설명형 데모"를 목표로 하지 않는다. 구현 대상은 **대학생이 캠퍼스 활동을 미션처럼 수행하고 그 기록을 남기는 개인용 캠퍼스 RPG형 기록 앱**의 웹 프로토타입이다.

핵심 방향:

- 메인 구현 대상은 루트 Next.js App Router 앱이다.
- `vibe-study-showcase`는 보조 버전으로만 유지한다.
- 화면은 **고정 휴대폰 비율** 안에서만 전환된다.
- 바깥 레이아웃은 앱을 소개하는 디자인 노트 역할만 수행한다.
- 실제 앱을 옮겨놓은 듯한 구조와 리듬을 우선한다.

---

## 1. Product Definition

산지니원정대는 **대학생이 캠퍼스 활동을 미션처럼 수행하고 그 기록을 남기는 개인용 캠퍼스 RPG형 기록 앱**이다.

이 앱의 핵심 목적은 다음 두 가지이다.

1. 캠퍼스 활동을 **미션 기반 구조**로 경험하도록 만든다.
2. 활동 기록을 **업적과 타임라인 형태로 아카이브**한다.

이 프로젝트는 프로덕션 서비스가 아니라 앱의 핵심 경험을 보여주는 **앱형 웹 프로토타입**이다.

## 3. Data and State Rules

모든 동작은 **local state only**로 구현한다.

허용:

```txt
useState
useEffect
useMemo
간단한 파생 계산
localStorage (선택)
```

금지:

```txt
backend
server actions 의존 로직
Firebase
Supabase
DB persistence
Redux, Zustand 등 복잡한 전역 상태
```

중요:

- 상태는 앱 내부 UI 흐름을 설명할 정도로만 단순하게 유지한다.
- 퀘스트, XP, streak, 튜토리얼 여부는 모두 세션 기준 local state로 관리한다.
- 같은 세션에서 완료한 퀘스트는 다시 완료할 수 없다.

권장 타입:

```ts
type AppTab = "home" | "map" | "record" | "memory";
type OverlayState = null | "tutorial" | "questPanel" | "celebration";
```

필수 상태:

```txt
currentTab
overlayState
selectedQuest
activeDialogueIndex
streakCount
completedToday
completedQuestIds
xpTotal
hasSeenTutorial
```

## 4. UI Structure

앱은 반드시 **고정 휴대폰 비율 프레임** 안에서 동작해야 한다.

필수 조건:

- 화면 비율은 `390 x 844`에 가까운 세로 비율 유지
- 탭이 바뀌거나 오버레이가 떠도 프레임 높이는 변하지 않음
- 스크롤이 필요하면 프레임 내부 콘텐츠 영역만 스크롤
- 데스크톱에서는 중앙의 휴대폰 프레임이 주인공이어야 함

바깥 설명 패널은 "기능 사용법"이 아니라 **왜 이런 앱을 디자인했는지** 설명하는 노트로 작성한다.

## 5. Information Architecture

앱 정보구조는 아래를 따른다.

```txt
홈
지도
기록
추억
```

반드시 하단 탭바를 사용한다.

### 홈

- 포스터형 홈이지만 정적 포스터 전시가 아니라 앱 홈 레이아웃으로 재구성
- 큰 히어로 영역
- 오늘의 미션 진행도
- 현재 레벨 / XP 게이지
- streak 요약
- 빠른 진입 CTA

### 지도

- 메인 플레이 화면
- 첫 진입 시에만 튜토리얼 오버레이 노출
- 지도 위 퀘스트 마커 클릭 가능
- 완료한 퀘스트는 비활성 + 완료 상태 유지

### 기록

- 완료한 퀘스트
- 누적 XP
- streak
- 오늘의 진행률
- 타임라인 또는 로그 형태 아카이브

### 추억

- 완료한 장소/미션을 카드형으로 보관
- 미완료 항목은 잠금 또는 흐림 처리

## 6. Tutorial and Quest Flow

첫 지도 진입 시에만 아래 튜토리얼 오버레이를 띄운다.

대사:

1.
```txt
안녕!
산지니 원정대에 온 걸 환영해!
```

2.
```txt
오늘도 캠퍼스에
새로운 모험이 기다리고 있어!
```

3.
```txt
오늘의 퀘스트를 확인해볼까?
```

튜토리얼 규칙:

- 배경 지도는 딤 처리
- `main-character` 자산이 말풍선으로 안내
- 타이핑 중 탭하면 문장 즉시 완성
- 마지막 대사 후 오버레이 종료 및 마커 활성화
- 이후 같은 세션에서는 튜토리얼을 다시 강제하지 않음

---

## 7. Quest and Reward Rules

최소 퀘스트 위치:

```txt
도서관
학생회관
금정산
```

퀘스트 완료 규칙:

- 모든 퀘스트 완료 시 축하 오버레이를 띄운다
- 화면은 어두워지고 산지니가 축하 말풍선을 보여준다
- XP 증가 연출을 동적으로 표시한다
- 첫 완료면 streak 보상도 함께 보여준다

XP 규칙:

- 모든 퀘스트 동일 XP
- 기본 XP는 `40`
- 레벨업 기준은 고정 `100 XP`
- 레벨 계산:

```ts
const level = Math.floor(xpTotal / 100) + 1;
```

같은 세션에서 같은 퀘스트는 다시 완료할 수 없다.

추가 규칙:

- 오늘의 3개 미션을 모두 완료하면 `오늘의 미션 전부 성공!` 메시지를 축하 오버레이에 추가
- 하루 전체 완료에 대한 추가 XP 보너스는 주지 않는다

## 8. Design Direction

톤:

```txt
모바일 앱처럼 자연스러운 구조
캠퍼스 RPG 기록 앱
밝고 친근한 색감
귀엽지만 설명형 데모처럼 보이지 않는 완성도
```

시각 가이드:

- 포스터 무드를 홈 히어로에 재해석
- 카드, 게이지, 탭바, 시트, 오버레이를 앱 UI처럼 구성
- 탭 전환과 오버레이는 가볍고 명확하게
- 지도/기록/추억이 서로 다른 화면으로 느껴지게 구분

피해야 할 것:

```txt
설명 패널 중심 레이아웃
화면마다 제멋대로 바뀌는 프레임 높이
포스터를 그대로 붙인 정적 데모 느낌
기업형 대시보드처럼 보이는 구성
```
