# HANDOFF.md

## Project: 산지니 원정대 앱형 웹 프로토타입

---

## 1. Overview

산지니원정대는 대학생이 캠퍼스 활동을 미션처럼 수행하고 그 기록을 남기는 **개인용 캠퍼스 RPG형 기록 앱**이다.

이 프로젝트의 목표는 작은 설명형 데모를 만드는 것이 아니라, **실제 앱을 옮겨놓은 듯한 웹 프로토타입**을 구현하는 것이다.

현재 구현 방향:

- 루트 Next.js App Router 앱이 메인 구현 대상
- `vibe-study-showcase`는 보조 쇼케이스로만 유지
- 앱은 고정 휴대폰 비율 안에서만 동작
- 화면 전환은 탭과 오버레이 중심
- 튜토리얼, 퀘스트, 기록, 추억, XP 성장감을 모두 앱 안에 포함

---

## 2. Product Goal

이 앱의 핵심 목적은 다음 두 가지이다.

1. 캠퍼스 활동을 **미션 기반 구조**로 경험하도록 만든다.
2. 활동 기록을 **업적과 타임라인 형태로 아카이브**한다.

사용자는 앱 안에서 다음 경험을 빠르게 이해해야 한다.

- 오늘의 미션을 확인한다
- 지도에서 퀘스트를 선택한다
- 퀘스트 완료 후 축하와 성장 보상을 받는다
- 기록/추억 탭에서 자신의 활동이 쌓이는 감각을 본다

---

## 3. App Information Architecture

앱은 아래 4개 탭을 기준으로 구성한다.

```txt
홈
지도
기록
추억
```

### 홈

- 포스터형 무드를 살린 앱 홈
- 오늘의 미션 진행도
- 현재 레벨 / XP
- streak 요약
- 지도 진입 CTA

### 지도

- 캠퍼스 지도 기반 메인 플레이 화면
- 퀘스트 마커 선택
- 튜토리얼 오버레이
- 퀘스트 바텀시트

### 기록

- 완료한 미션 로그
- XP 누적
- 오늘의 진행률
- streak 표시

### 추억

- 완료한 미션/장소의 카드형 아카이브
- 미완료 항목은 잠금 상태 유지

---

## 4. Phone Shell Rules

앱은 반드시 **고정 휴대폰 비율 프레임**으로 보여야 한다.

가이드:

- 기준 비율: `390 x 844`
- 상태가 바뀌어도 프레임 높이 변화 금지
- 프레임 내부 화면만 전환
- 필요한 경우 내부 콘텐츠만 스크롤

데스크톱에서는 폰 프레임이 중심이고, 옆 패널은 기능 설명이 아니라 디자인 의도 설명을 보여준다.

추천 문구:

> 산지니원정대는 대학생이 캠퍼스 활동을 미션처럼 수행하고 그 기록을 남기는 개인용 캠퍼스 RPG형 기록 앱이다.
>
> 이 앱의 핵심 목적은 캠퍼스 활동을 미션 기반 구조로 경험하도록 만들고, 활동 기록을 업적과 타임라인 형태로 아카이브하는 것이다.

## 6. Tutorial Flow

첫 지도 진입 시에만 튜토리얼 오버레이를 띄운다.

대사:

1.
```txt
안녕!
산지니 원정대에 온 걸 환영해!
```

2.
```txt
오늘도 캠퍼스에
새로운 모험이 기다리고 있어!
```

3.
```txt
오늘의 퀘스트를 확인해볼까?
```

튜토리얼 종료 후:

- 딤 제거
- 지도 마커 활성화
- 이후 같은 세션에서는 다시 자동 재생하지 않음

---

## 7. Quest and Reward Logic

최소 퀘스트:

- 도서관
- 학생회관
- 금정산

보상 규칙:

- 모든 퀘스트 완료 시 축하 오버레이 노출
- XP `+40`
- XP 게이지 상승 애니메이션
- 첫 완료면 streak 표시 추가
- 같은 세션에서 같은 퀘스트는 재완료 불가
- 3개 미션 전체 완료 시 `오늘의 미션 전부 성공!` 메시지 추가

레벨 규칙:

```txt
100 XP마다 레벨 1 상승
```

예시:

```txt
0~99 XP = 레벨 1
100~199 XP = 레벨 2
200~299 XP = 레벨 3
```

## 8. Implementation Notes

우선순위:

1. 고정 휴대폰 비율 앱 셸
2. 하단 탭바와 4개 화면
3. 지도 튜토리얼 오버레이
4. 퀘스트 완료 축하 + XP 시스템
5. 기록/추억 탭 아카이브화

중요:

- 서버 없이 local state only
- 정적 export 가능해야 함
- 이미지 자산은 앱 UI의 일부로 재구성

---

## 9. Risks

주의할 점:

- "데모 카드 모음"처럼 보이면 안 됨
- 화면마다 프레임 높이가 달라지면 앱 느낌이 깨짐
- 설명 패널이 메인보다 눈에 띄면 방향이 틀어짐
- 완료 보상이 첫 퀘스트에만 보이면 성장감이 약함
