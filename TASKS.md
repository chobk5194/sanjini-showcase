# TASKS.md

## Project: 산지니 원정대 앱형 웹 프로토타입

이 문서는 현재 구현 방향에 맞춘 작업 순서를 정리한다.

이번 패스의 메인 구현 대상은 **루트 Next.js 앱**이다.

---

## Phase 1. App Shell Reset

목표:

- 루트 Next 앱을 앱형 구조로 재정의
- 고정 휴대폰 비율 프레임 도입
- 내부 화면 전환 구조 준비

작업:

- `GameShell`을 고정 비율 폰 프레임 구조로 재작성
- 상단 상태 영역과 하단 탭바 구성
- 콘텐츠 영역은 내부 스크롤 가능 구조로 정리

완료 조건:

- 어떤 화면에서도 폰 프레임 높이가 변하지 않는다

---

## Phase 2. Types and Mock Data

목표:

- 앱형 상태 모델과 보상 시스템 타입 정의

작업:

- `AppTab`, `OverlayState` 타입 추가
- `Quest`에 `xpReward` 추가
- `PlayerProgress` 또는 동등한 진행 구조 추가
- 모든 퀘스트에 XP `40` 설정

완료 조건:

- 탭, 오버레이, XP 관련 상태를 타입으로 안전하게 표현할 수 있다

---

## Phase 3. Home Screen

목표:

- 정적 포스터가 아니라 앱 홈처럼 보이는 첫 화면 구현

작업:

- 홈 히어로 영역 구성
- 오늘의 미션 진행률 카드
- 레벨 / XP 게이지
- streak 요약
- 지도 진입 CTA

완료 조건:

- 홈 탭이 앱의 첫 화면으로 자연스럽게 보인다

---

## Phase 4. Map + Tutorial

목표:

- 지도 메인 플레이 화면과 첫 진입 튜토리얼 구현

작업:

- 지도 탭 구현
- 퀘스트 마커 3개 배치
- 첫 지도 진입 시 딤 오버레이 + 캐릭터 튜토리얼
- 마지막 대사 후 마커 활성화

완료 조건:

- 첫 지도 진입에서만 튜토리얼이 재생되고 이후에는 바로 탐험 가능

---

## Phase 5. Quest + Celebration + XP

목표:

- 퀘스트 선택과 완료 보상 시스템 구현

작업:

- 퀘스트 바텀시트 구현
- 완료 시 축하 오버레이 노출
- XP `+40` 증가 애니메이션
- 레벨 게이지 갱신
- 첫 완료 시 streak 카드 표시
- 3개 미션 전체 완료 시 전체 완료 메시지 추가

완료 조건:

- 모든 퀘스트 완료에 대해 축하 오버레이가 나온다
- 같은 퀘스트는 다시 완료할 수 없다

---

## Phase 6. Record Tab

목표:

- 활동이 쌓이는 로그형 화면 구현

작업:

- 완료한 퀘스트 목록
- 획득 XP 요약
- streak / 진행률 카드
- 시간순 또는 상태순 타임라인 카드 구성

완료 조건:

- 기록 탭이 단순 더미가 아니라 앱의 성장 로그처럼 보인다

---

## Phase 7. Memory Tab

목표:

- 장소와 미션을 추억 카드처럼 아카이브

작업:

- 완료한 항목은 추억 카드로 표시
- 미완료 항목은 잠금 또는 흐림 처리
- 기록 탭과 다른 감성의 레이아웃 적용

완료 조건:

- 추억 탭이 기록 탭과 분명히 다른 역할로 보인다

---

## Phase 8. Desktop Presentation

목표:

- 데스크톱에서도 앱 프로토타입처럼 보이게 정리

작업:

- 왼쪽 패널을 디자인 노트로 재작성
- 기능 설명 대신 디자인 목적과 제품 정의를 표시
- 모바일에서는 패널을 축약 또는 숨김 처리

완료 조건:

- 폰 프레임이 시각적 중심으로 유지된다

---

## Phase 9. Validation

확인 항목:

- 고정 비율 폰 프레임 유지
- 홈/지도/기록/추억 탭 전환
- 첫 지도 튜토리얼 1회 재생
- 퀘스트 완료마다 축하 오버레이
- XP / 레벨 업데이트
- 첫 완료 streak 표시
- 전체 미션 완료 메시지
- 정적 export 호환 유지
